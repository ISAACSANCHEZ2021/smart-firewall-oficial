Configurar Firewall

❌ Manual (tedioso)

Configurar el firewall manualmente en un servidor puede ser lento y propenso a errores.

sudo ufw allow 22/tcp sudo ufw allow 80/tcp sudo ufw allow 443/tcp sudo ufw allow 3000/tcp sudo ufw allow 8080/tcp sudo ufw allow 25565/tcp

• Debes saber qué puertos usan los servicios
• Debes abrirlos uno por uno
• Si olvidas uno el servicio no funcionará
• Puede volverse muy tedioso en servidores grandes

✅ Smart Firewall

Smart Firewall detecta automáticamente los servicios activos y te pregunta si deseas permitirlos.

sudo ./smart-firewall.sh ⚠ Servicio detectado Programa : nginx Puerto : 80 ¿Permitir en firewall? (s/n): s ✅ Puerto 80/tcp permitido

• Detecta servicios automáticamente
• No necesitas recordar puertos
• Configuración rápida y segura
• Ideal para Ubuntu Server

Un firewall estilo Windows para Linux

¿Te gustaría tener un firewall en tu servidor Linux que funcione parecido al de Windows? Es decir, que cuando una aplicación necesite abrir un puerto el sistema te muestre un mensaje preguntando si quieres permitirlo.

Smart Firewall trae una experiencia similar a Ubuntu Server. El script escanea automáticamente los servicios activos en el sistema, detecta qué puertos necesitan y luego te pregunta si deseas permitirlos en el firewall.

🪟 Firewall típico (Windows)

Cuando una aplicación intenta acceder a la red aparece una ventana preguntando si deseas permitirla.

Windows Security Alert Application: nginx Port: 80 Allow access? [ Allow ] [ Block ]

🐧 Smart Firewall

Smart Firewall hace algo similar pero optimizado para servidores Linux sin interfaz gráfica.

⚠ Servicio detectado Programa : nginx Puerto : 80 Protocolo: tcp ¿Permitir en firewall? (s/n):

En lugar de buscar manualmente los puertos de cada aplicación o configurar reglas una por una, Smart Firewall escanea los servicios activos usando herramientas del sistema como ss y te muestra exactamente qué programa quiere usar qué puerto.
