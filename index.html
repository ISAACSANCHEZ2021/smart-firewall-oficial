Configure Firewall

❌ Manual (tedious)

Manually configuring the firewall on a server can be slow and error-prone.

sudo ufw allow 22/tcp sudo ufw allow 80/tcp sudo ufw allow 443/tcp sudo ufw allow 3000/tcp sudo ufw allow 8080/tcp sudo ufw allow 25565/tcp

• You must know what ports the services use
• You must open them one by one
• If you forget one the service will not work
• Can become very tedious on large servers

✅ Smart Firewall

Smart Firewall automatically detects active services and asks you if you want to allow them.

sudo ./smart-firewall.sh ⚠ Service detected Program : nginx Port : 80 ¿Allow in firewall? (s/n): s ✅ Port 80/tcp allowed

• Detects services automatically
• You don't need to remember ports
• Quick and secure setup
• Ideal for Ubuntu Server

A Windows-style firewall for Linux

¿You would like to have a firewall on your Linux server that works similar to that of Windows? That is, when an application needs to open a port the system shows you to message asking if you want to allow it.

Smart Firewall brings a similar experience to Ubuntu Server. The script automatically scans active services on the system, detects what ports they need and then asks you if you want to allow them on the firewall.

🪟 Typical firewall (Windows)

When an application tries to access the network, a window appears asking if you want to allow it.

Windows Security Alert Application: nginx Port: 80 Allow access? [ Allow ] [ Block ]

🐧 Smart Firewall

Smart Firewall does something similar but optimized for Linux servers without graphic interface.

⚠ Service detected Program : nginx Port : 80 Protocol: tcp ¿Allow in firewall? (S/n):

Instead of manually looking up ports for each application or configuring rules one by one, Smart Firewall scans the active services using tools from the system like ss and shows you exactly which program wants to use which port.
